2. Tables¶

Tables come in four different flavors:

  1. BasicTables allow for any combination of objects and unfold deterministically
  2. SimpleTables are like BasicTables but allow only one ball at a time
  3. NoisyTables are SimpleTables that utilize stochastic physics from Smith & Vul (2013)
  4. GravityTables are SimpleTables in which objects fall towards the bottom

2.1. BasicTable¶

class BasicTable(dims, closed_ends, background_cl, def_ball_rad, def_ball_cl, def_pad_len, def_wall_cl, def_occ_cl, def_pad_cl, active, soffset, defscreen)¶

Initializes and returns a BasicTable object.

Parameters:
  • dims – A tuple (x,y) of table dimensions in pixels
  • closed_ends – A list of direction constants to determine which sides of the table should be walled off. Defaults to [LEFT, RIGHT, BOTTOM, TOP]
  • background_cl – Color of the table background. Defaults to WHITE
  • def_ball_rad – Default radius (in px) for objects.Ball objects added to the table. Defaults to 20
  • def_ball_cl – Default color of objects.Ball objects added to the table. Defaults to BLUE
  • def_pad_len – Default length (in px) of objects.Paddle objects added to the table. Defaults to 100
  • def_wall_cl – Default color of objects.Wall and objects.AbnormWall objects added to the table. Defaults to BLACK
  • def_occ_cl – Default color of objects.Occlusion objects added to the table. Defaults to GREY
  • def_pad_cl – Default color of objects.Paddle objects added to the table. Defaults to BLACK
  • active – Defines the initial activity of the table. See the BasicTable.activate() and BasicTable.deactivate() methods. Defaults to True
  • soffset – A tuple (x,y) of the offset from the parent Surface to place this table for drawing. If None (default), places the table in the middle of the surface
  • defscreen – A pygame.Surface object on which to place the BasicTable. If None (default) and a pygame window is open, defaults to the display surface

2.1.1. Adding Objects¶

BasicTable.addBall(initpos, initvel, rad, color, elast)¶

Adds a ball to the table, centered at an (x,y) coordinate defined in initpos, with a velocity vector (vx,vy) given to initvel. If not defined, radius and color are set to the table defaults, and elasticity is set to 1.

BasicTable.addWall(upperleft, lowerright, color, elast)¶

Adds a rectangular wall to the table, with upperleft and lowerright coordinates given. If not defined, color is set to the table default, and elasticity is set to 1.

BasicTable.addAbnormWall(vertexlist, color, elast)¶

Adds a non-rectangular wall to the table. Vertices are given as a list of (x,y) tuples in vertexlist, and must be convex and counterclockwise. If not defined, color is set to the table default, and elasticity is set to 1.

BasicTable.addOcc(upperleft, lowerright, color)¶

Adds a rectangular occluder to the table, with upperleft and lowerright coordinates given. Balls can pass through occluders, but cannot be seen through them. If not defined, color is set to the table default.

BasicTable.addGoal(upperleft, lowerright, onreturn, color)¶

Adds a rectangular goal to the table, with upperleft and lowerright coordinates given. When a ball hits a goal the table will no longer unfold in time, and BasicTable.step() calls will return the value defined in onreturn. If a color is not given, the goal will be invisible.

BasicTable.addPaddle(p1, p2, padlen, padwid, hitret, active, acol, iacol, pthcol, elast)¶

Adds a paddle to the table, that can move between endpoints p1 and p2. The paddle must be verticle or horizontal. If padlen is not defined, the length of the paddle itself is defined by table defaults. The padwid argument defines the thickness of the paddle. Paddles can act as goals if hitret is set - when a ball contacts the paddle, then BasicTable.step() calls will return hitret. If active is set to False, the paddle cannot be moved. Colors can be set for active paddles (acol), inactive paddles (iacol), and the background path the paddle can travel (pthcol).

2.1.2. Simulating¶

BasicTable.step(t, maxtime)¶

Simulates the table forward by t seconds. t defaults to 1/50. and should be evenly slicable into milliseconds to avoid step rounding.

If nothing happens during this step, this returns None. If a ball hits a goal, the function will return the goal’s onreturn value. If maxtime is given and the table has been running for more than that value (in seconds), this returns the constant TIMEUP.

2.1.3. Drawing¶

BasicTable.draw(stillshow)¶

Draws the table onto the parent surface, and returns a pygame.Surface object with just the table drawing on it. If stillshow is set to True, the balls will be shown through the objects.Occlusion objects.

BasicTable.demonstrate(screen, timesteps, retpath, onclick, maxtime)¶

Simulates the path of the table forward until a ball hits a goal or time expires, then returns the time it took until expiration.

Parameters:
  • screen – A pygame.Surface object to draw on
  • timesteps – The time (in seconds) between animation frames. Defaults to 1/50.
  • onclick – Can take in a function that takes a BasicTable as an argument, then calls that function on this table whenever the mouse is clicked (for debugging)
  • maxtime – The maximum time (in seconds) the demonstration will run without hitting a goal. If not given, will run (potentially) forever.
BasicTable.makeMovie(moviename, outputdir, fps, removeframes, maxtime)¶

NOTE: requires ffmpeg to function

Like demonstrate(), but makes the demonstration into a movie file.

Parameters:
  • moviename – Name of the movie file
  • outputdir – The directory to place the movie in (defaults to current directory)
  • fps – The frames per second to run the movie at (defaults to 20)
  • removeframes – If set to False, keeps the individual frames around after the movie is created
  • maxtime – Like BasicTable.demonstrate(), the movie will run until a ball hits a goal, or until it has run for maxtime seconds

2.1.4. Utilities¶

BasicTable.activate()¶

Sets the table to be active, so the BasicTable.step() command steps the world forward

BasicTable.deactivate()¶

Sets the table to be inactive, so that the ball does not move when BasicTable.step() is called (however, paddle positions, etc. will update)

BasicTable.mostlyOcc(ball)¶

Tests whether a specific objects.Ball is at least half covered by an occluder; returns True if so, False otherwise

BasicTable.mostlyOccAll()¶

Returns a list of boolean values asking whether each of the BasicTable.balls is mostly covered.

BasicTable.fullyOcc(ball)¶

Tests whether a specific objects.Ball is completely covered by an occluder; returns True if so, False otherwise

BasicTable.fullyOccAll()¶

Returns a list of boolean values asking whether each of the BasicTable.balls is fully covered.

BasicTable.activatePaddle()¶
BasicTable.deactivatePaddle()¶
BasicTable.togglePaddle()¶

Set the paddle to active, inactive, or switches its activity

BasicTable.getRelativeMousePos()¶

Returns the mouse position offset from the upper-left-hand corner of the table as an (x,y) tuple.

BasicTable.assignSurface(surface, offset)¶

Reassigns the parent surface to surface, with a given offset

BasicTable.set_timestep(ts)¶

Sets the minimum time between physical simulation steps to ts. This will by default be set to 1ms. Do not change this value lightly: higher values will cause the simulation to run faster, but run the risk of creating wonky, non-physical events if too coarse

2.1.5. Object Access¶

BasicTable.balls¶

Returns a list of object.Ball objects attached to the table

BasicTable.walls¶

Returns a list of object.Wall and object.AbnormWall objects attached to the table

BasicTable.occludes¶

Returns a list of object.Occlusion objects attached to the table

BasicTable.goals¶

Returns a list of the object.Goal objects attached to the table

BasicTable.goalrettypes¶

Returns a list of the possible values that can be returned when a ball hits a goal on this table. Use the function physicsTable.Constants.getConst() to interpret these values.

BasicTable.paddle¶

Returns the object.Paddle object on this table if it exists; otherwise, returns None

2.2. SimpleTable¶

SimpleTable objects are very similar to BasicTable objects with one exception: they cannot contain more than one ball. All functions are the same as BasicTable except where noted below

class SimpleTable(dims, closed_ends, background_cl, def_ball_rad, def_ball_cl, def_pad_len, def_wall_cl, def_occ_cl, def_pad_cl, active, soffset, defscreen)¶

Identical to BasicTable, but returns a SimpleTable instead

SimpleTable.addBall(initpos, initvel, rad, color, elast, dispwarn)¶

As BasicTable.addBall() except that if an objects.Ball already exists on the table, it will be overwritten. If dispwarn is set to True, you will be warned in the console if this happens.

SimpleTable.mostlyOcc()¶
SimpleTable.mostlyOccAll()¶
SimpleTable.fullyOcc()¶
SimpleTable.fullyOccAll()¶

As the same named methods from BasicTable except none take arguments and they all return a single boolean representing the occlusion state of the one ball on the SimpleTable

2.3. NoisyTable¶

NoisyTable objects are SimpleTable objects that do not unfold deterministically. Instead each ball is placed with uncertainty in the position and velocity, and the physics unfolds noisily at each time step and whenever the ball bounces. Noise is defined by four parameters:

  • perr: When a ball is placed, its location is slightly uncertain, and thus gets displaced from its real value. perr is the standard deviation of a 2-D Gaussian defining the noise in that position
  • kapv: When a ball is placed, its direction of motion is also uncertain. Its velocity thus is drawn from a vonMises distribution centered around the actual direction with concentration parameter kapv
  • kapm: At each point in time that the ball’s motion can be ‘jittered’ by redrawing it from a vonMises distribution centered around the current direction with concentration parameter kapm. NOTE: if the physics timesteps are changed with BasicTable.set_timestep(), the impact of this parameter will change
  • kapb: Whenever the ball hits a wall, it can bounce at an odd angle. Therefore its direction is redrawn from a vonMises distribution centered around the direction it should travel with concentration parameter kapv

See Smith & Vul (2013) for further description of these noise parameters. All parameters default to the values used in that paper

Typically, NoisyTable objects are created by “making” a SimpleTable noisy through the makeNoisy() function:

makeNoisy(table, kapv, kapb, kapm, perr)¶

Takes in a SimpleTable and uncertainty parameters and returns a NoisyTable. If uncertainty parameters are not given, they default to the values of Smith & Vul (2013)

If constructed de novo, NoisyTable objects are created like parent tables

class NoisyTable(dims, kapv, kapb, kapm, perr, closed_ends, background_cl, def_ball_rad, def_ball_cl, def_pad_len, def_wall_cl, def_occ_cl, def_pad_cl, active, soffset, defscreen)¶

All parameters are the same as BasicTable, with additional parameters as described above.
